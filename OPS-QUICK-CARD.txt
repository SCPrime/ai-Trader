📊 PRE-LIVE CHECKLIST
─────────────────────────────────────────────────────────────────────────────
  ☐ Kill-switch test (enable → try live execute → expect 423 error)
  ☐ LIVE_TRADING env var correct (false for safety, true for live window)
  ☐ Duplicate detection working (run smoke test)
  ☐ Redis connected (check /api/health for "redis":{"connected":true})
  ☐ Backend logs clear (no errors in Render → Logs)

🛡️ SAFETY LAYERS
─────────────────────────────────────────────────────────────────────────────
  1. LIVE_TRADING=false (Render env) → All orders become dry-run
  2. Kill-Switch=true → Blocks live execute even if LIVE_TRADING=true
  ⚠️  BOTH must be green for live orders to execute

📞 QUICK TROUBLESHOOTING
─────────────────────────────────────────────────────────────────────────────
  Proxy 404?        → Vercel → Settings → Root Directory = "frontend"
  Proxy 500?        → Vercel → Env Vars → Check BACKEND_API_BASE_URL, API_TOKEN
  Backend 403?      → Render → ALLOW_ORIGIN must match Vercel domain
  Duplicate fails?  → Check backend logs; verify requestId sent in body
